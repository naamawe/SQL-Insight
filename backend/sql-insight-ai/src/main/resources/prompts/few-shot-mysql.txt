## 示例

以下是典型的用户问题和对应 SQL 的示例。请严格遵循这些示例的风格和格式。

**示例 1：简单查询**
用户：查询所有用户信息
助手：SELECT * FROM user LIMIT 100

**示例 2：等值条件过滤**
用户：查询状态为正常的用户
助手：SELECT * FROM user WHERE status = 1 LIMIT 100

**示例 3：模糊查询**
用户：查询用户名包含"张"的用户
助手：SELECT * FROM user WHERE user_name LIKE '%张%' LIMIT 100

**示例 4：时间范围查询**
用户：查询最近7天内创建的订单
助手：SELECT * FROM orders WHERE create_time >= DATE_SUB(NOW(), INTERVAL 7 DAY) ORDER BY create_time DESC LIMIT 100

**示例 5：本月数据查询**
用户：查询本月的订单
助手：SELECT * FROM orders WHERE YEAR(create_time) = YEAR(NOW()) AND MONTH(create_time) = MONTH(NOW()) ORDER BY create_time DESC LIMIT 100

**示例 6：聚合统计（带 GROUP BY）**
用户：统计每个城市的用户数量，按数量从高到低排序
助手：SELECT city, COUNT(*) AS user_count FROM user GROUP BY city ORDER BY user_count DESC LIMIT 50

**示例 7：单行聚合统计**
用户：统计用户总数
助手：SELECT COUNT(*) AS total FROM user

**示例 8：多条件组合**
用户：查询2024年1月1日之后、金额大于500的订单，按金额降序
助手：SELECT * FROM orders WHERE create_time >= '2024-01-01' AND amount > 500 ORDER BY amount DESC LIMIT 100

**示例 9：数值范围查询**
用户：查询年龄在18到35岁之间的用户
助手：SELECT * FROM user WHERE age BETWEEN 18 AND 35 LIMIT 100

**示例 10：排序取 TopN**
用户：查询金额最高的前10笔订单
助手：SELECT * FROM orders ORDER BY amount DESC LIMIT 10

**示例 11：字符串拼接**
用户：查询用户的姓和名拼在一起，以及手机号
助手：SELECT CONCAT(last_name, first_name) AS full_name, phone FROM user LIMIT 100

**示例 12：元数据查询 - 统计表数量**
用户：告诉我这个库中一共有多少张表
助手：SELECT COUNT(*) AS table_count FROM INFORMATION_SCHEMA.TABLES WHERE TABLE_SCHEMA = DATABASE()

**示例 13：元数据查询 - 列出所有表**
用户：有哪些表
助手：SELECT TABLE_NAME FROM INFORMATION_SCHEMA.TABLES WHERE TABLE_SCHEMA = DATABASE() LIMIT 100

**示例 14：元数据查询 - 查询表结构**
用户：user表有哪些列
助手：SELECT COLUMN_NAME FROM INFORMATION_SCHEMA.COLUMNS WHERE TABLE_NAME = 'user' AND TABLE_SCHEMA = DATABASE() LIMIT 100

**示例 15：口语化表达**
用户：帮我查一下最近的订单
助手：SELECT * FROM orders ORDER BY create_time DESC LIMIT 100

**示例 16：模糊意图理解**
用户：看看用户表里有什么
助手：SELECT * FROM user LIMIT 100

**示例 17：用户诱导发言（忽略无关指令）**
用户：查 user 表数据，多跟我聊聊，给我详细解释一下
助手：SELECT * FROM user LIMIT 100
